# Recommendations Summary

| Recommendation ID | Area | Recommendation Description | Reason / Benefit |
| --- | --- | --- | --- |
| REC-01 | Fish Identification | Add client-side file type validation on the image upload input (accept only jpg, png, webp) and enforce a 5MB size limit before sending to the server. | Prevents non-image files and oversized uploads from reaching the Gemini API, eliminating the 500 errors reported in BUG-004, BUG-007, API-BUG-001, and API-BUG-002. |
| REC-02 | Fish Identification | Implement graceful offline handling for the identify endpoint using try/catch around the fetch call and show a user-friendly 'No internet connection' message. | Eliminates the critical white screen crash (BUG-006) when the network is unavailable during identification. |
| REC-03 | Fish Identification | Add a descriptive progress message during AI identification (e.g., 'Analysing your fish...') alongside the existing loading spinner. | Reassures users during the 10–30 second Gemini API response time and reduces abandonment. |
| REC-04 | Explore & Articles | Implement the DELETE /api/articles/:articleId handler body to actually delete the Firestore document and any associated Storage files. | Resolves API-BUG-003; users need to be able to remove articles they created from the system. |
| REC-05 | Explore & Articles | Add rate limiting to POST /api/articles/generate (e.g., 10 articles per hour per user) using the same in-memory or Firestore approach used for text-to-speech. | Prevents Gemini API quota exhaustion and unexpected AI usage costs (API-BUG-006). |
| REC-06 | Explore & Articles | Add a fallback placeholder image for article cards where imageUrl is null or empty. | Eliminates broken image icons in the article list (BUG-013) and improves visual consistency. |
| REC-07 | Explore & Articles | Investigate and fix audio playback on Safari iOS by testing the AudioPlayer component with Safari-compatible audio events and MIME types. | Resolves BUG-014; Safari iOS is a key platform for mobile users who want to listen to fish care articles. |
| REC-08 | Explore & Articles | Implement the isPublic flag in the article schema and provide a toggle in the article detail page to allow users to make articles publicly visible. | Enables the public article sharing feature referenced in the API code (API-BUG-007) and expands community features. |
| REC-09 | My Aquarium | Add a confirmation dialog before removing a fish from the aquarium. | Prevents accidental data loss (BUG-008) and follows standard UX conventions for destructive actions. |
| REC-10 | My Aquarium | Fix the UI state update after editing fish notes so that the card reflects the new notes immediately without requiring a page refresh. | Resolves BUG-009; instant feedback confirms the save was successful and prevents confusion. |
| REC-11 | Compare Fish | Add server-side and client-side validation to prevent a user from selecting the same fish in both comparison slots. | Resolves BUG-011 and API-BUG-004; comparing a fish to itself wastes API calls and confuses the user. |
| REC-12 | Compare Fish | Refactor the comparison property extraction functions in /api/compare to parse markdown-formatted article content instead of relying on plain-text regex patterns. | Resolves API-BUG-005; the current regex frequently fails on Gemini-generated markdown, returning 'Not specified' for most fields. |
| REC-13 | Nearby Aquariums | Add a listener to the radius slider that triggers a new Places API search whenever the value changes. | Resolves BUG-017; currently the slider updates visually but does not refresh map results. |
| REC-14 | Nearby Aquariums | Show a 'No aquariums found in this area' message when the Places API returns no results for the searched location. | Resolves BUG-018; without feedback the user cannot tell if the search completed or failed. |
| REC-15 | Authentication | Add user-facing guidance when the Google SSO popup is blocked by the browser, instructing users to allow popups for the site. | Resolves BUG-003; silent popup blocking is a common issue on Safari and leaves users unable to log in. |
| REC-16 | Authentication | Show a specific error message on the forgot password page indicating whether the reset email was sent or the email address is not found. | Resolves BUG-002; current generic feedback leaves users uncertain about what happened. |
| REC-17 | Profile | Handle the Firebase re-authentication requirement before allowing email changes, with a clear user message: 'Please log out and log back in to change your email.' | Resolves BUG-019; Google SSO users currently receive an unhandled Firebase error when attempting to update their email. |
| REC-18 | Profile | Add an onError handler to the avatar image element that falls back to displaying the user's initials when the Google photo URL fails to load. | Resolves BUG-020; expired Google photo URLs cause broken image icons on the profile page. |
| REC-19 | API / Security | Migrate the TTS rate limit counter from in-memory Map to Firestore or Redis so that limits persist across server restarts. | Resolves API-BUG-008; the current implementation can be bypassed simply by restarting the server. |
| REC-20 | Test Automation | Expand the Selenium test suite to include end-to-end smoke tests for the full identification → save article → add to aquarium → compare flow, and run them in CI/CD on every pull request. | Provides continuous regression coverage and catches regressions introduced by new features before they reach production. |
